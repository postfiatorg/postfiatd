Detailed Description

ExclusionManager maintains an in-memory cache of validator exclusion lists and efficiently determines which addresses are excluded by consensus.

An address is considered excluded if at least minCONSENSUS_PCT (67%) of UNL validators have that address on their exclusion list.

Note: The cache is built based on the current UNL from the validator list. Only validators in the UNL can contribute to the consensus exclusion, though the SetAccount transaction currently uses ValidatorVoteStats as a proxy for validator status due to transaction context limitations.
